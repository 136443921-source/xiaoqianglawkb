## 🎯 核心理念

传统知识管理的问题：存得多、用得少、找不到。

**LLM Wiki 的解决方案**：
- **Raw 层**（`raw/`） - 存放原始资料，AI 只读不改
- **Wiki 层**（`wiki/`） - AI 编译后的结构化知识
- **Schema 层**（`AGENTS.md`） - 工作手册，定义规则和格式

**优势**：
- ✅ 知识沉淀：一次编译，持续复用
- ✅ 自动关联：`[[wikilinks]]` 自动建立知识网络
- ✅ 高效查询：基于结构化 Wiki，而非临时搜索碎片
- ✅ 持续进化：使用过程中，新结论可写回 Wiki

## 🚀 快速开始

### 第一步：采集资料

将法律资料放入 `raw/` 目录对应子目录：

**桌面端**（推荐）：
1. 安装 Obsidian Web Clipper 浏览器插件
2. 看到好文章时，点击插件图标
3. 选择保存到 `kb-wiki/raw/articles/` 目录

**移动端**（微信文章）：
1. 在微信中看到好文章，分享到 IMA 知识库
2. 在桌面端打开 IMA，导出为 Markdown
3. 保存到 `kb-wiki/raw/articles/` 目录

**现有资料导入**：
- 从 Obsidian 导入：复制笔记文件到 `raw/notes/` 目录
- 从 flomo 导入：导出 CSV，让 WorkBuddy 转换为 Markdown

详见《数据采集指南.md》

### 第二步：编译知识

在 WorkBuddy 会话中输入：

```
阅读当前项目 raw/ 中新增的文件，按照项目根目录中的 AGENTS.md 的规范，
将它们编译到 wiki/ 目录中，并同时更新 index.md 和 log.md 文件。
```

WorkBuddy 会：
1. 读取 `raw/` 中的文件
2. 生成结构化的 Wiki 页面到 `wiki/` 对应子目录
3. 使用 `[[wikilinks]]` 建立页面间关联
4. 更新 `index.md` 和 `log.md`

### 第三步：查询知识

在 WorkBuddy 中直接提问，例如：

```
根据知识库内容，解释"合同无效"的构成要件，并列出法条依据和相关案例。
```

WorkBuddy 会：
1. 先读 `index.md`，判断哪些页面相关
2. 加载对应的 Wiki 页面
3. 基于结构化知识回答

### 第四步：定期维护

每周执行一次健康检查：

```
请对当前知识库进行一次健康检查，检查以下内容：
1. 有没有矛盾的信息
2. 有没有孤立的页面（没有任何其他页面引用它）
3. 有没有断掉的 [[wikilinks]]（指向不存在的页面）
4. 有没有过时的法条引用

检查完成后，生成一份健康报告，并给出修复建议。
```

## ⚖️ 律师专用特性

本知识库针对律师工作场景定制：

1. **法条引用规范**：
   - 必须包含：法规全称、条款号、条款内容摘要
   - 注意新旧法衔接（如《合同法》已废止，应为《民法典》第XXX条）

2. **案例价值评估**：
   - 优先标注：最高人民法院指导性案例
   - 其次标注：高级人民法院典型案例
   - 再次标注：中级人民法院有参考价值的案例

3. **法律概念时效性**：
   - 在概念页中注明依据的最新司法解释或指导案例

4. **利益冲突检查**：
   - 涉及具体案件时注意脱敏
   - 不在 Wiki 中记录当事人完整信息（除非是公开的指导性案例）

详见 `AGENTS.md` 中的"律师专用规则"章节。

## 🔧 已连接的法律工具

当前 WorkBuddy 已连接以下法律相关 MCP 服务：

- ✅ **华宇元典法律数据**（`yuandian-mcp`）- 法律数据库查询
- ✅ **IMA 知识库**（`ima-mcp`）- 移动端采集中转
- ✅ **腾讯文档**（`tencent-docs`）- 文档管理
- ✅ **企业微信**（`wecom`）- 协作沟通

利用这些工具，可以实现：
- 从 IMA 自动同步采集的内容到 `raw/` 目录
- 查询华宇元典数据库，补充知识库内容
- 将 Wiki 页面同步到腾讯文档，方便分享
