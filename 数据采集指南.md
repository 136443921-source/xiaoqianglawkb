## 🖥️ 桌面端采集

### 方案：Obsidian Web Clipper 浏览器插件

**适用场景**：在桌面浏览器中看到有价值的法律文章、判决书、学术论文等

**步骤**：

1. **安装插件**
   - 打开 Chrome/Edge 浏览器
   - 访问 Chrome 网上应用店
   - 搜索 "Obsidian Web Clipper"
   - 点击"添加至 Chrome"

2. **配置插件**
   - 点击浏览器工具栏中的 Obsidian Web Clipper 图标
   - 设置保存位置：`kb-wiki/raw/` 目录
   - 建议按类型分子目录：
     - `raw/articles/` - 法律文章
     - `raw/judgments/` - 判决书
     - `raw/papers/` - 学术论文
     - `raw/regulations/` - 法律法规
     - `raw/notes/` - 个人笔记

3. **使用插件**
   - 在浏览器中打开想要保存的网页
   - 点击 Obsidian Web Clipper 图标
   - 选择保存位置（子目录）
   - 点击"保存"
   - 插件会自动将网页内容抓取为 Markdown 格式，保存到指定目录

**优点**：
- 快速、简单
- 自动转换格式
- 保留原文格式和链接

**注意事项**：
- 部分网站可能有反爬虫机制，导致抓取失败
- 建议抓取后检查 Markdown 文件，确保内容完整
- 对于重要文档，建议同时保存 PDF 版本到 `raw/` 对应目录

## 📱 移动端采集

### 问题：微信文章无法直接抓取

在手机微信中看到好文章，无法直接用 Obsidian Web Clipper 抓取。

### 方案：IMA 知识库中转到 raw/ 目录

**适用场景**：微信文章、朋友圈文章、其他移动端内容

**步骤**：

#### 第一阶段：移动端收藏到 IMA

1. **安装 IMA 知识库 App**
   - iOS：App Store 搜索"IMA"
   - Android：应用市场搜索"IMA"

2. **在微信中收藏文章**
   - 打开微信文章
   - 点击右上角"..."菜单
   - 选择"复制链接"或"在 Safari 中打开"（iOS）
   - 粘贴链接到 IMA App，或直接在 IMA 中收藏

3. **或者直接分享到 IMA**
   - 部分版本支持直接分享到 IMA 知识库
   - 具体操作见 IMA App 内说明

#### 第二阶段：从 IMA 同步到 raw/ 目录

**方案 A：手动同步（推荐用于初期）**

1. 在桌面端打开 IMA 知识库
2. 找到收藏的文章
3. 导出为 Markdown 或 PDF
4. 保存到 `kb-wiki/raw/` 对应子目录

**方案 B：自动同步（使用 WorkBuddy 定时任务）**

1. **配置 IMA MCP 连接**
   - 确保 WorkBuddy 已连接 IMA 知识库（已在你的配置中看到 `ima-mcp` 已连接）
   - 测试连接：在 WorkBuddy 中执行 `mcp__ima-mcp__get_knowledge_base_list` 查看知识库列表

2. **创建定时任务**
   - 在 WorkBuddy 中创建自动化任务
   - 设置每周或每天执行一次
   - 任务内容：
     ```
     检查 IMA 知识库中的新增内容，将以下内容同步到 kb-wiki/raw/ 目录：
     1. 过去7天新增的文章
     2. 标签为"待整理"的内容
     
     同步时：
     - 保存为 Markdown 格式
     - 文件名格式：文章标题_YYYYMMDD.md
     - 保存到 raw/articles/ 目录
     - 在文件开头添加元数据：
       ---
       title: 文章标题
       source: IMA 知识库
       collected_at: YYYY-MM-DD
       tags: [标签1, 标签2]
       ---
     ```

3. **测试定时任务**
   - 先手动执行一次，检查同步效果
   - 确认文件格式和保存位置正确
   - 调整任务参数（如同步频率、文件过滤规则等）

**优点**：
- 移动端采集通道打通
- 自动化同步，减少手动操作
- IMA 知识库可作为临时收集箱

**注意事项**：
- IMA 知识库的 API 可能有调用限制，注意控制同步频率
- 同步前先检查 raw/ 中是否已有相同内容，避免重复
- 定期清理 IMA 知识库中的已同步内容

## 📂 现有资料导入

### 场景：已有 Obsidian 或 flomo 笔记

如果你之前已经在 Obsidian 或 flomo 中积累了法律笔记，可以将它们导入到知识库。

#### 从 Obsidian 导入

1. **复制文件**
   - 打开原 Obsidian 仓库目录
   - 选择要导入的 Markdown 文件
   - 复制到 `kb-wiki/raw/notes/` 目录

2. **批量处理（可选）**
   - 如果文件很多，可以让 WorkBuddy 帮忙批量添加元数据
   - 示例命令：
     ```
     读取 raw/notes/ 目录中的所有 Markdown 文件，在每个文件开头添加以下元数据：
     ---
     title: 文件名（不含扩展名）
     source: 原有 Obsidian 笔记
     imported_at: 2026-06-27
     tags: [笔记]
     ---
     如果文件已有元数据（--- 格式），则合并而不是覆盖。
     ```

#### 从 flomo 导入

1. **导出 flomo 数据**
   - 打开 flomo App 或网页版
   - 进入"设置" → "数据导出"
   - 选择导出格式（推荐 CSV 或 Markdown）
   - 导出所有笔记

2. **转换格式（如需要）**
   - 如果导出的是 CSV，需要转换为 Markdown
   - 可以让 WorkBuddy 帮忙转换：
     ```
     读取 flomo_export.csv 文件，将每条笔记转换为独立的 Markdown 文件，
     保存到 raw/notes/ 目录。
     文件名格式：笔记标题_YYYYMMDD.md（如果笔记无标题，使用笔记前20个字符）
     在每个文件开头添加元数据：
     ---
     title: 笔记内容前50字符
     source: flomo
     created_at: 笔记创建时间
     tags: [从 flomo 标签提取]
     ---
     ```

3. **保存到 raw/ 目录**
   - 将转换后的 Markdown 文件保存到 `kb-wiki/raw/notes/` 目录

## 📏 数据采集原则

### 宁缺毋滥

**原则**：只采集有价值的内容

**判断标准**：
- ✅ **应该采集**：
  - 权威来源（最高人民法院、高级人民法院、知名法学院校）
  - 经典案例（指导性案例、典型案例）
  - 前沿问题（新型案件、疑难问题）
  - 个人深度思考（不是简单的摘抄）

- ❌ **不应该采集**：
  - 质量低下的文章（错误观点、逻辑混乱）
  - 过时的法律法规（已被废止或修订）
  - 重复内容（已在知识库中的相似内容）
  - "先存了再说"的内容（没有明确价值）

### 元数据完整

**原则**：每个原始资料都应该有基本的元数据

**最低要求**：
```markdown
---
title: 文档标题
source: 来源（网址/书名/期刊名）
collected_at: 采集日期
tags: [标签1, 标签2]
---

# 文档标题

（正文内容）
```

**推荐添加**：
- `author` - 作者
- `published_at` - 发布日期
- `type` - 类型（ article/judgment/paper/regulation）
- `relevance` - 相关度（高/中/低）

### 分类清晰

**原则**：采集时就按类型分类，便于后续编译

**推荐目录结构**：
```
raw/
├── articles/      - 法律文章、评论
├── judgments/     - 判决书、裁定书
├── papers/        - 学术论文
├── regulations/   - 法律法规、司法解释
├── notes/         - 个人笔记、思考
└── temp/          - 临时收集、待分类
```

## 🔄 数据采集工作流

### 完整流程

```
1. 发现有价值内容
   ├─ 桌面端：浏览器中看到
   ├─ 移动端：微信/其他 App 中看到
   └─ 现有资料：Obsidian/flomo 中已有

2. 采集到 raw/ 目录
   ├─ 桌面端：Obsidian Web Clipper
   ├─ 移动端：IMA 中转 → 手动或自动同步
   └─ 现有资料：导入并转换格式

3. 检查数据质量
   ├─ 元数据是否完整？
   ├─ 内容是否清晰？
   └─ 分类是否正确？

4. 标记待编译
   └─ 在 log.md 中记录："待编译：raw/articles/XXX.md"

5. 触发编译
   └─ 在 WorkBuddy 中执行编译命令（见下节）
```

## 🛠️ 下一步：编译知识

数据采集完成后，需要编译到 Wiki 层。

**编译命令模板**（在 WorkBuddy 中执行）：

```
阅读当前项目 raw/ 中新增的文件，按照项目根目录中的 AGENTS.md 的规范，
将它们编译到 wiki/ 目录中，并同时更新 index.md 和 log.md 文件。

本次编译范围：
- raw/articles/ 中的 X 个文件
- raw/judgments/ 中的 Y 个文件

请按照 AGENTS.md 中的"页面类型规范"生成对应页面，并建立 [[wikilinks]] 关联。
```

## 📞 问题排查

### Obsidian Web Clipper 抓取失败

**可能原因**：
- 网站有反爬虫机制
- 网页内容动态加载（JavaScript 渲染）
- 插件配置错误

**解决方案**：
- 尝试手动复制网页内容，粘贴到新建的 Markdown 文件中
- 或使用浏览器的"打印为 PDF"功能，保存后再手动转换为 Markdown

### IMA 同步失败

**可能原因**：
- IMA MCP 连接断开
- API 调用次数超限
- 网络问题

**解决方案**：
- 检查 WorkBuddy 的 connector-status，确认 `ima-mcp` 为 connected
- 如果断开，尝试重新连接
- 减少同步频率，避免触发 API 限制

### 导入的文件格式混乱

**可能原因**：
- 原文件格式不标准
- 转换工具处理不当

**解决方案**：
- 让 WorkBuddy 帮忙批量格式化
- 或手动检查并修正格式错误的文件
